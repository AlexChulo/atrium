mismatches

infix fun <T : CharSequence> Expect<T>.mismatches(expected: Regex): Expect<T> (source)

Expects that the subject of the assertion (a CharSequence) mismatches the given expected.

In contrast to containsNot.regex it does not look for a partial match but for an entire match.

Exceptions

AssertionError - Might throw an AssertionError if the assertion made is not correct.

Return
This assertion container to support a fluent API.
